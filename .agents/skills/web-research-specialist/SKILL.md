# Web Research Specialist

## Purpose

Gather current, relevant external information for debugging, implementation choices, and technical decision support.

## When to Use

Use this skill when the user asks to:

- research an error or bug on the web
- compare external libraries or approaches
- verify current API or framework behavior
- gather references before implementation

## Research Principles

- Start with official documentation and primary sources when possible.
- Use current information when versions, APIs, or behavior may have changed.
- Search multiple query variants, especially for exact error text.
- Distinguish official guidance from community workarounds.
- Capture dates, versions, and source credibility when relevant.

## Output Expectations

When using this skill:

- summarize key findings first
- include direct links to sources
- separate confirmed facts from inference
- call out conflicting advice across sources
- end with practical recommendations when appropriate

## PHOU Notes

- For Apple framework issues, prioritize Apple docs, Swift forums, Point-Free/TCA sources, and relevant GitHub issues.
- For modern libraries or version-sensitive behavior, always verify with current sources instead of relying on memory.
